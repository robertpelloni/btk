## Interpretation

- The restored historical Script baseline already aligns with a large portion of the current WebKit-embedded JavaScriptCore tree, which validates the recovery-first strategy.
- The remaining drift is concentrated in legacy subtrees and files expected by historical Script but absent from the current WebKit snapshot, especially older `pcre`, `wrec`, and selected runtime-era files.
- This means Stage A should continue as an adaptation exercise, not a provenance exercise: the module is back, but its JavaScriptCore contract must now be narrowed or translated to match the modern embedded tree.
