## Minimum Script manifest interpretation

### Downstream/public compatibility minimum
BTK will need, at minimum, public Script headers/types sufficient for declarative public headers that reference `QScript*` symbols. The exposed pressure is concentrated around:
- `qml/qdeclarativecomponent.h`
- `qml/qdeclarativeengine.h`
- `qml/qdeclarativecontext.h`
- `qml/qdeclarativeexpression.h`

### Declarative compiler bring-up minimum
In addition to public Script APIs, `CsDeclarative` needs private bridge support, especially:
- `qscriptdeclarativeclass_p.h`
- `QScriptDeclarativeClass`

### Practical reading

- Public compatibility alone is not enough. Even if BTK provided a minimal `qscriptvalue.h`, declarative internals would still fail without the bridge layer.
- The first viable `CsScript` restoration pass should therefore be split into two checkpoints: public header compatibility first, then private declarative bridge support for `CsDeclarative` compilation.
