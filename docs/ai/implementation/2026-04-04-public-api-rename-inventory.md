## Migration Interpretation

- `CsCore`, `CsGui`, `CsNetwork`, and sibling library target names are good candidates for staged BTK target aliasing before deep renames.
- Annex namespaces (`CsString`, `CsPointer`, `CsSignal`, `CsLibGuarded`) are API-sensitive and should be migrated via compatibility wrappers/typedefs rather than abrupt replacement.
- `cs_` file prefixes are widespread enough that file-system renames should be deferred until after alias-based API stabilization.
- Remaining `CopperSpice` branding is still extensive and should be attacked in phases: package/config/docs first, public API second, file names/install layout last.
